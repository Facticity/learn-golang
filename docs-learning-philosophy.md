# Learning Philosophy and Assumption Policy

## Assumption Policy
This course intentionally avoids assuming prior Go knowledge beyond basic programming.
Each module should answer:
1. What are we doing?
2. Why this approach instead of alternatives?
3. What tradeoffs exist?
4. How does this appear in production teams?

## Quality bar for a "complete" repo
A complete repo needs:
- runnable code
- tests
- operational workflows
- explicit rationale
- beginner-friendly FAQ coverage
